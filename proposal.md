## 一、产品定位

**一句话**：青灯的个人阅读札记——青灯之下，黄卷之间，为值得重读的字句留一笔朱批。

**调性**：静谧、克制、东方书卷气。不是技术教程站，是「一个人在灯下读书写字」的私人刊物。

**差异化**：当下中文博客多为「卡片+代码块」的技术流或「杂志感」的设计流；青灯黄卷走「宋体+泛黄纸+朱砂印」的文人札记路线，靠气质而非信息密度取胜。

## 二、目标受众

- **主**：喜欢长文阅读、在意排版细节的读者（书评/随笔/技术反思的受众）
- **次**：前端/设计圈里欣赏「反 AI 审美」的同行
- **非目标**：追热点、看速食内容的人

## 五、核心功能（发散，参考优秀博客）

### P0 必做
1. **Markdown 驱动**（已有）— frontmatter + 正文，构建期注入
2. **RSS / Atom feed** — 自动生成 `feed.xml`，博客订阅刚需
3. **文章 TOC 目录** — 长文右侧浮动目录，宋体编号「一、二、三」
4. **阅读进度条**（已有）+ ** estimated reading time**（已有）
5. **脚注 / 旁注** — `[^1]` 语法的脚注，hover 展开为旁注小字（古书批注感）

### P1 推荐
6. **归档页** — 卷册式时间线，「卷一·2026春」分组
7. **标签页** — 朱砂方印式标签，点击进该标签文章列表
8. **相关文章** — 文末「延伸卷册」3 篇（同标签/同系列）
9. **系列文章** — frontmatter 加 `series`，支持「上一卷/下一卷」连续阅读
10. **文章页增强**：首字下沉（已有）、落款（文末朱砂日期+印）

### P2 可选
11. **搜索** — PageFind（构建期索引，无后端）
12. **字号/行距调节** — 长读友好（A- A+ 按钮）
13. **返回顶部** — 朱砂小按钮
14. **图片灯箱** — 点击放大（如果用图）

## 六、内容模型（frontmatter 扩展）

现有：
```yaml
title / deck / date / read / tags / featured
```

建议扩展：
```yaml
series: reading-notes      # 系列，支持连续阅读
updated: 2026-07-10        # 更新日期（显示「修订于」）
draft: false               # 草稿不构建
cover: /assets/xxx.png     # 封面图（可选）
footnotes: true            # 启用脚注渲染
```

## 七、设计增强创意（发散，贴合书卷气）

这些是「让青灯黄卷有别于普通博客」的独特点子，挑你喜欢的做：

1. **旁注（margin notes）** — 正文侧边栏的小字批注，像古书页边的朱笔批注。`[^note]` 自动渲染为旁注，窄屏折叠成脚注。
2. **卷轴式阅读进度** — 把顶部进度条改成右侧竖向细线，像卷轴徐徐展开。
3. **印章式标签** — 标签渲染成朱砂方印（朱红方块+白字），点击进标签页。
4. **卷册归档** — 归档页按「卷一/卷二」+ 年份分组，像古籍目录。
5. **落款** — 文章末尾：朱砂日期 + 小方印「青」字，像作者签章。
6. **目录用「卷」编号** — TOC 用「一、二、三」中文数字，而非 1/2/3。
7. **引文块用古籍版式** — blockquote 加引号「」+ 朱砂左边框，像古籍夹批。
8. **首字下沉用朱砂** — 当前是青，可改朱砂强化「朱批」感。
